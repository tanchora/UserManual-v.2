3. Создание проекта

  1. Первым шагом работы над моделью является создание проекта в Eclipse. Нам нужен специальный вид проектов - “Modeling Project”.
  2. Затем, создадим модель разрабатываемого сайта или импортируем в проект готовый шаблон модели. Файл модели имеет расширение *.vinci. В Nasdanika Tool Suit есть специальный мастер для создания таких моделей.
  3. Далее, для проекта необходимо настроить Viewpoint (“Точку зрения”), что даст возможность в дальнейшем работать с диаграммой связей Action.
  4. Кроме того, целесообразно будет открыть модель в специальном редакторе, чтобы было удобнее работать.

Рассмотрим этот процесс подробнее.

1. Новый проект 2. Новая Vinci модель 3. Импорт готовой Vinci модели 4. Выбираем Viewpoint

1. В области Model Explorer щелкнуть правой кнопкой мыши и создать новый проект.

1.1 ПКМ -> New -> Modeling Project

В результате у нас должна появиться папка проекта, в которой есть всего два файла: ProjectDependendencies и representations.aird.

1.2 Если по каким-то причинам вам не удалось создать Modeling Project, вы можете сначала создать простой проект, а затем конвертировать его в Modeling Project.

Папка General -> раскрываем папку -> выбираем Project -> нажимаем Next

->

1.3 Присваиваем проекту имя (латиницей, без пробелов и специальных символов)

1.4 В поле Location выбираем место, в котором будет располагаться проект. Предварительно убираем галочку Use default location. Если предполагается публиковать проект на GitHab - располагаем его в репозиторий.

1.5 У нас не будет связанных проектов, поэтому это диалоговое окно мы пропускаем

2. Конвертируем простой проект в Modeling Project

2.1 На корневой папке проекта ПКМ -> Configure -> Convert to Modeling Project

в результате в проекте появляются новая папочка и файл representations.aird - очень важный и нужный.

3. Создаем новую Vinci модель.

3.1 Выделяем корневой каталог (папку) проекта -> ПКМ -> New -> Other
3.2 Прокручиваем вниз, находим папку Nasdanika -> раскрываем папку -> выделяем Vinci Model -> Next
3.3 Присваиваем модели имя -> Next

–

3.4 Выбираем объект (Model Object) для которого будем строить модель. Это будет объект Page

4. Ура! Модель создана и открылась для редактирования справа от области Model Explorer.

Но мы хотим использовать другой, более удобный редактор, поэтому мы закроем редактор модели, и затем:

4.1 Вернемся в область Model Explorer -> выделим нашу модель -> ПКМ

4.2 Open With -> Vinci Model Sirius Editor

3. Импортируем готовую Vinci модель.

5. Последним шагом мы выбираем Viewpoint для нашего проекта

5.1 Выделяем корневой каталог проекта -> ПКМ -> Viewpoints Selection

5.2 Ставим галочку, выбираем Viewpoint Vinci

Проект готов к работе!
